Overview of what the program does

This wrapper code allows you to open a .fits image and create a galfit model for objects in it, using ds9 regions to massively streamline the process.

It uses a base TUI interface to run commands, which then let you open images in ds9, where you can place regions for each object you want to model. The regions will then be automatically optimized as a galfit model. For experienced galfit users, further tweaking of the regions to enhance the model and an easy to use added text editor for the corresponding config file let you quickly produce quality models.

For each file you open, a corresponding output will be saved inside a gf_out folder in your home directory. Any progress you make in your model (including config files, the psf, constraints, and the finished model) will be saved and loaded back in whenever you open the same image back up in the wrapper program, allowing you to easily pick back up modeling with the same object.

Installation

Currently tested for mac OS and WSL on windows. For both options, only bash is supported!

Quickstart guide ->

  • Make sure ds9 is installed and callable from bash
  • Download galfit executable
  • Create conda environment and install the required packages
  • Clone repository
  • From repository directory, run galfit_wrapper.py

In depth installation guides for Mac and Windows ->

Installing on Mac:

1) Make sure you have xquartz installed

  • This allows applications to open from bash
  • Can install here

2) Install ds9:

  • There are two versions of ds9: you need the X11 version for the wrapper!
  • Download the latest X11 installation for your Mac OS here
  • Once installed, make sure ds9 is callable from bash. You can do this as follows:
  • Copy the downloaded ds9 exec file in your local bin directory – this should be at /usr/local/bin

3) Download galfit executable

  • You can download the file here - make sure to get the Mac OS version.
  • Move the downloaded file to wherever you would like to keep it, and make sure to save the path to where you saved it (you will need this later).

4) Create conda environment and install the required packages

  • Create new environment
  • $ conda create -n galfit_wrapper
  • Activate environment (you will need to run this command everytime you start the code)
  • $ conda activate galfit_wrapper
  • Install required packages
  • $ conda install pip
  • $ pip install numpy
  • $ pip install astropy
  • $ pip install --upgrade [--user] pyds9
  • $ pip install pyregion
  • $ pip install sep

5) Clone repository

  • In terminal, use cd command to move to whatever directory you would like to have the wrapper code in.
  • Clone the repository with command
  • $ git clone https://github.com/paxsonswierc/galfit_wrapper.git

6) Using the wrapper for the first time

  • Everything should be set up now! To use the program, enter
  • $ python galfit_wrapper.py
  • The first time you run the program, it will ask for your path to the galfit file that you downloaded earlier. Type that in and hit enter without any extra spaces.
  • If you messed up and entered the wrong path, just delete the file named path_config.txt and run the wrapper again. It will prompt you to enter the path again.
  • For further instructions on running, see our example run and the commands list!
  • When you are finished, just enter quit – After you have left the program you can deactivate your conda environment with
  • $ conda deactivate
  • Anytime you want to run the program again, remember to cd to the directory where you cloned the program and activate your conda environment beforehand!

Guide to output files

  • To find outputs, go to your home directory and search for a folder named gf_out
  • Inside this folder, there will be corresponding folders for every object you have worked on using the code. Each folder will share the same name as fits file you were working with.
  • Navigate inside the folder of the object you would like to get files for. Each file will start with the name of the object, as follows. Models you are likely to want to download are bolded

General Files:

  • filename.fits – a copy of the fits target image you were working with
  • zero_point.txt – the zero point you provided
  • rgb_info.txt – scaling and path info for if you loaded other files for color image

        PSF Files:

  • filename_psf_config.txt – your config file for the psf
  • filename_psf_mask.fits – mask file for your psf model
  • filename_psf.fits – the galfit output fits for your psf model
  • filename_psf_model.fits – just the model frame for you psf model

        General (Sersic) Model Files:

  • filename_config.txt – the config file for your model of the target
  • filname_constraint.txt – the constraint file for your model
  • filename_mask.fits – mask file for your model
  • filename_model.fits – galfit output model file (with all 4 frames)
  • filename_model_prod.fits – output galfit model file for if you have run sersic produce config

TUI Commands

(list; what it does; intricacies; what it can’t do)

*We refer to your general model as sersic, since most modeling we support is with sersic profiles

**Aliases for each command in parentheses

  • help (?; h)
  • Will list all the available commands
  • quit (exit)
  • Exits the wrapper, also closing any instances of DS9
  • target list (target l; tl; ls)
  • Lists the names and last saved dates of the original target FITS, the PSF config, the PSF model, sersic config, sersic model, and constraints on the sersic
  • target visualize (target v; tv)
  • Opens the original target FITS in DS9
  • target visualize rgb (target v rgb; tv rgb)
  • Prompts user to enter target files in red, green, blue order - it is recommended to use files from ~/gf_out/target_filename/target_filename.fits
  • change zero point (change zpt; cz)
  • You are asked the zeropoint of the image in the first run, but you can change this at any time with this command
  • psf create (psf c; pc)
  • Creates the PSF from scratch.
  • Opens the target file, and prompts the user to find a star to put a circle around, then a box around that.
  • Circle regions will create galfit moffat profiles.
  • Can place multiple circle regions for multiple moffat components.
  • Box regions will create the area over which the model is run.
  • Note exclude (mask area) and background (don’t subtract component) can be added at any time. The typical and perfect PSF would have one circle centered at a point, and a larger box surrounding that point.
  • psf visualize (psf v; pv)
  • View the output image block from the galfit optimization, blinking the frames in blank, target, model, residual order.
  • psf flags (psf f; pf)
  • Any flags that were output during the galfit run process for the most recent run with a produced output image block will print.
  • psf upload (psf u; pu)
  • If you have a one-band psf already created for this target, you can upload it.
  • sersic create config (sersic cc; scc)
  • Creates new model
  • Will walk you through the process of adding regions to represent model components.
  • Ellipse regions are sersic profiles
  • Point regions are psf profiles
  • Box region is area over which model is run
  • After optimizing, you may choose to save the model, undo the optimization and edit the regions you had placed, or save changes and make new edits to the optimized model.
  • Further loops will allow for manual edits in the our text editor and the addition of an automatically created constraint file
  • sersic edit config (sersic ec; sec)
  • Open up an already created config, and edit the regions/manually edit the file in our text editor.
  • You may optimize it after.
  • sersic add constraint (sersic add c; sac)
  • Automatically generates a constraint file for your model
  • sersic remove constraint (sersic rm c; src)
  • Removes any existing constraint file loaded for your model
  • sersic optimize config (sersic oc; soc)
  • Optimizes a created/uploaded config file
  • Enters into a loop of options to undo changes, continue edits on the optimized model, or save the optimized model.
  • sersic produce config (sersic pc; spc)
  • Allows you to save a version of your final model with objects removed.
  • Click on a region and hit b to remove them from the output model.
  • If you remove all but a single modeled object, the residuals will have everything except that object left, allowing you to remove objects from a well modeled image!
  • sersic visualize (sersic v; sv)
  • Opens up optimized sersic model in ds9, flipping through the galfit output frames.
  • sersic visualize regions (sersic vr; svr)
  • Opens up the regions corresponding to objects in the config, in ds9.
  • Not for editing, just for visualization.
  • sersic visualize rgb (sersic v rgb; sv rgb)
  • If you have visualized the target in color, allows you to also view your model in rgb (if you have created models for other bands).
  • sersic flags (sersic f; sf)
  • Any flags that were output during the galfit run process for the most recent run with a produced output image block will print.
  • sersic upload config (sersic uc; suc)
  • Allows you to upload a config text file.
  • Will automatically convert it to a region representation so you can make edits and optimize it in the wrapper.
  • sersic upload model (sersic um; sum)
  • Allows you to upload a completed model, purely for visualization purposes.
  • sersic upload constraint (sersic ucst; sucst)
  • Allows you to upload a constraint file for your model. Will be overwritten if you later choose to auto generate a constraint.

EXAMPLE FULL RUN (for a 3-band image)

Step 1: Download and Prepare FITS Files

  • Files must only contain one band, so if you have 3-band image, then split it up
  • Prepare Header with any relevant information, complete any transformations in data if necessary
  • For example: if data is downloaded from DECaLS, it needs to be multiplied by 10000 for a realistic zeropoint, and header items “EXPTIME” and “GAIN” should be added/set to 1 (secret command “mult fits” in program)
  • Have zeropoint of each band calculated
  • Keep filenames as short as possible to avoid running into overflow issues with galfit

Step 2: Open File using galfit runner

  • If using conda, first activate environment
  • Run “python3 galfit_runner.py” from directory containing the code file
  • It will first prompt you to locate the file, which will allow for a new directory to be created in ~/gf_out directory
  • It will then ask for the zeropoint of the image, enter, but can change anytime using “change zero point”
  • Since we’re dealing with 3 band images, it makes sense to set up all directories and zeropoints, for convenience purposes; repeat the above steps for the other bands
  • Now that directories are created, can run the program without needed to reinput FITS file with “python3 galfit_runner.py ~/gf_out/filename/filename.fits

Step 3: Visualize the Target

  • You want to choose the best filter for modelling, which may be the middle filter, or another choice (“r” filter band is a good idea for grz triplet)
  • It makes sense to look at the image before doing anything else to get bearings! Use “target visualize”, this will open up a DS9 window for convenient viewing
  • “target list” will let you know what files already exist in the directory
  • “target visualize rgb” is a helpful tool if you are working with 3-band images, for again to get bearings
  • Be sure to input the files in the correct order, and specifically use the files in /gf_out in case changes are made

Step 4: Create PSF

  • “psf create” will allow you to create psf component
  • It will first prompt you to place a circle region around the PSF
  • Then, it will prompt you to place a box region around the PSF
  • NOTES: can add exclude region at any point, should try to center box around PSF as good as possible, make sure circle of PSF is also centered on star
  • It will optimize the galfit model, save the result, and also save the 1-band model from the 4-band model.
  • “psf visualize” will allow you to see the results of the most recently saved PSF config/model
  • “psf flags” will print the flags that galfit originally gave
  • “psf upload” allows you to skip the creating process, if you already have a one-band FITS image of the PSF for this image
  • MASK: _psf_mask.fits will always be created, but blank unless exclude regions are made. Will not be deleted but is overwritten like config; so most recent model was created using the existing mask file

Step 5: Do the actual modelling

  • “sersic create config” allows you to create the model from scratch
  • It will first prompt you to place ellipse regions for sersic components
  • Then point regions for PSF regions
  • Then box around the model
  • NOTES: can add exclude regions at any point, can add “do not include in final model” regions also using properties->background
  • Optimize galfit model, save the result, display the 4-band result
  • Asks you to:
  • 1: save the model and config
  • You are happy with the model! The residuals look good to you, and the model is what you want it to be. This will keep the saved config file, and the 4-band output from galfit.
  • 2: edit the output config of this model
  • The model is close to what you want. There are a couple edits you would like to make, but you think the galfit optimization makes sense, and you want to edit FROM the galfit optimized regions
  • 3: reset from last stage and edit last config
  • The model isn’t what you want, and it’s worse than what you had before. This allows you to go back to the regions you had placed before the galfit model ran for the most recent iteration.
  • 2 or 3 will ask you the following:
  • Edit config file manually
  • “Yes” will bring up a text editor, you can make changes here and save
  • Equivelently, you can open the file in your own text editor, and make any changes before clicking save on this file (or equivalently, “enter” when offered to edit)
  • Edit the current regions
  • DO NOT DELETE REGIONS
  • You can change shapes of regions, move them around, add new regions you wish to constraint (if you want to add constraint)
  • Asks for constraint, will auto-generate:
  • +/- 1 pixel, +/- 4 magnitude, +/-10% on other values on all reigons
  • Can add more regions, that will not be constrained (equivalent to adding regions before, if you didn’t add a constraint file)
  • Will run galfit again, and repeat process
  • If galfit crashes, the loop is broken, and you should either start over or edit regions from run before it crashed
  • Once loop is broken, you can use “sersic edit config” to edit the currently existing config file to optimize your model
  • Same as above
  • “sersic optimize config” will simply run galfit for an existing config file, skipping above steps with region editing
  • “Sersic visualize” will allow you to visualize 4-band target, model, and residual
  • At any point, you can add a constraint to the existing config file, or remove a constraint. This constraint is the same one that is offered during optimizing a galfit model
  • “sersic flags” will print out flags from the most recent run of galfit
  • You can upload config, 4-band model files, and constraints too.

  • When you are happy with the model, run “sersic add constraint” so you can use it in the next model.

Step 6: Repeat for Other Bands

  • The same exact process. But we already did a lot of the work, so we can use upload functionalities
  • PSFs must be done from scratch, follow above steps
  • After happy with PSF, can
  • Upload config
  • Will plot regions on the image you’re working with
  • Upload constraint
  • Either edit the config, or directly optimize
  • Follow steps as normal, until happy with the model; feel free to remove constraint at any time
  • Can now visualize the model in all bands! Sv rgb
